# /commit

변경된 파일을 분석해서 Conventional Commits 규칙에 따라 커밋을 수행한다.

## 진행 순서

1. `git status`로 변경 파일 목록 확인
2. `git diff` (staged + unstaged)로 실제 변경 내용 파악
3. 변경 내용을 분석해 아래 규칙에 따라 커밋 메시지 작성
4. 관련 파일을 staging하고 커밋 실행

## 커밋 메시지 규칙

### 형식

```
<type>(<scope>): <description>

[optional body]
```

### type 선택 기준

| type | 사용 상황 |
|------|-----------|
| `Feat` | 새로운 기능 추가 |
| `Fix` | 버그 수정 |
| `Refactor` | 기능 변경 없는 코드 구조 개선 |
| `Style` | CSS, 디자인, 포맷 변경 (동작 무관) |
| `Chore` | 빌드, 설정, 패키지, 도구 변경 |
| `Docs` | 문서, 주석, README 변경 |
| `Test` | 테스트 코드 추가/수정 |
| `Perf` | 성능 개선 |
| `Revert` | 이전 커밋 되돌리기 |

### scope (선택)

변경 범위를 나타내는 소문자 단어. 예: `auth`, `nav`, `ui`, `api`, `design`

파일이 여러 영역에 걸쳐 있으면 생략한다.

### description

- 50자 이내의 명령형 현재 시제
- 한국어 또는 영어 (프로젝트 기존 커밋 언어를 따름)
- 마침표 없음

### body (선택)

- 변경 이유나 맥락이 필요할 때만 작성
- 72자 줄바꿈

## 금지 사항

- `Co-Authored-By:` 트레일러 추가 금지
- `Generated by Claude` 등 AI 도구 언급 금지
- `--no-verify` 플래그 사용 금지

## 커밋 실행

```bash
git commit -m "$(cat <<'EOF'
<type>(<scope>): <description>
EOF
)"
```

body가 있을 경우:

```bash
git commit -m "$(cat <<'EOF'
<type>(<scope>): <description>

<body>
EOF
)"
```

## 예시

```
Feat(auth): 로그인 유지 체크박스 추가
Fix(nav): 모바일 메뉴 닫힘 버그 수정
Style(design): 다크 테마를 라이트 테마로 전환
Chore: pnpm-lock.yaml 업데이트
Refactor(auth): validateUser 에러 처리를 findUnique로 교체
Docs: AGENTS.md에 디자인 시스템 룰 추가
```
